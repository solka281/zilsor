# Руководство по управлению боссами рейдов

## Для администратора

### Доступ к функциям
1. Используйте команду `/admin` в боте
2. В админ-панели найдите кнопки:
   - **🐉 Запустить рейд** - запуск рейда вручную
   - **➕ Создать босса** - создание нового босса
   - **✏️ Редактор боссов** - редактирование существующих боссов

### Создание нового босса

#### Процесс создания (7 шагов):
1. **Название** - имя босса (1-50 символов)
   - Пример: "Повелитель огня"
   
2. **Уровень** - сложность босса (1-120)
   - Влияет на отображение, не на механику
   
3. **Базовое HP** - стартовое здоровье (10,000-1,000,000)
   - Будет увеличиваться на 5,000 за каждую победу
   
4. **Описание** - краткое описание (1-200 символов)
   - Пример: "Древний повелитель стихии огня"
   
5. **Золото** - награда золотом (100-10,000)
   - Делится между игроками пропорционально урону
   
6. **Кристаллы** - награда кристаллами (5-100)
   - Делится между игроками пропорционально урону
   
7. **Опыт** - награда опытом (100-10,000)
   - Делится между игроками пропорционально урону

#### Автоматические параметры:
- **Изображение**: автоматически генерируется имя файла
  - "Повелитель огня" → `raid_повелитель_огня.jpg`
- **ID**: присваивается автоматически
- **Счетчик побед**: начинается с 0

### Редактирование босса

#### Доступные параметры:
- **📝 Название** - изменить имя босса
- **⭐ Уровень** - изменить уровень (1-120)
- **❤️ HP** - изменить базовое HP (10,000-1,000,000)
- **📝 Описание** - изменить описание (1-200 символов)
- **🖼️ Изображение** - изменить имя файла изображения
- **💰 Золото** - изменить награду золотом (100-10,000)
- **💎 Кристаллы** - изменить награду кристаллами (5-100)
- **✨ Опыт** - изменить награду опытом (100-10,000)
- **🗑️ Удалить босса** - удалить босса (с подтверждением)

#### Ограничения:
- Нельзя удалить босса, который используется в активном рейде
- Изменения не влияют на текущий активный рейд
- Счетчик побед можно сбросить только через базу данных

### Прогрессивная система HP

#### Как работает:
1. Босс создается с базовым HP (например, 75,000)
2. После каждой победы игроков HP увеличивается на 5,000
3. Формула: `Текущее HP = Базовое HP + (Количество побед × 5,000)`

#### Примеры прогрессии:
- **0 побед**: 75,000 HP
- **1 победа**: 80,000 HP (+5,000)
- **5 побед**: 100,000 HP (+25,000)
- **10 побед**: 125,000 HP (+50,000)

#### Отображение в игре:
```
❤️ HP: 80,000/80,000
🏆 Побед: 1 (+5,000 HP)
```

### Управление изображениями

#### Требования к изображениям:
- **Формат**: JPG (предпочтительно)
- **Размер**: до 10 МБ
- **Разрешение**: 800x600, 1024x768 или 1280x720
- **Расположение**: папка `images/raids/`

#### Именование файлов:
- Автоматическое: `raid_название_босса.jpg`
- Ручное: любое имя, заканчивающееся на `.jpg`

#### Добавление изображения:
1. Создайте или найдите подходящее изображение
2. Переименуйте в соответствии с именем босса
3. Поместите в папку `images/raids/`
4. При необходимости измените имя файла через редактор

### Запуск рейдов

#### Автоматический запуск:
- Новый рейд создается автоматически через 2 часа после завершения предыдущего
- Использует случайного босса из базы данных

#### Ручной запуск:
- Используйте кнопку "🐉 Запустить рейд" в админ-панели
- Можете выбрать уровень босса или случайного
- Отменяет текущий активный рейд

### Мониторинг системы

#### Логи:
- Все действия администратора логируются
- Создание/редактирование/удаление боссов записывается
- Запуск рейдов фиксируется

#### Статистика:
- Количество побед каждого босса
- Текущее HP с учетом прогрессии
- История рейдов в базе данных

## Для игроков

### Что изменилось:
- Боссы становятся сильнее с каждой победой
- В меню рейдов отображается количество побед и бонус HP
- Могут появляться новые боссы, созданные администратором

### Отображение прогрессии:
```
🐉 ВЫБОР БОССА

🌪️ Повелитель ветра
⭐ Уровень: 1
❤️ HP: 80,000/80,000
🏆 Побед: 1 (+5,000 HP)
⏰ Доступен сейчас! (118 мин)

💰 Награды (делятся по урону):
• 1000 золота
• 20 алмазов  
• 1000 опыта
```

## Техническая информация

### База данных:
```sql
CREATE TABLE raid_bosses (
  id INTEGER PRIMARY KEY AUTOINCREMENT,
  name TEXT NOT NULL UNIQUE,
  level INTEGER NOT NULL,
  base_hp INTEGER NOT NULL,
  times_defeated INTEGER DEFAULT 0,
  image TEXT,
  description TEXT,
  rewards TEXT,
  created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);
```

### API функции:
- `raids.createBossTemplate(bossData, callback)`
- `raids.updateBoss(bossId, field, value, callback)`
- `raids.updateBossRewards(bossId, rewardType, value, callback)`
- `raids.deleteBoss(bossId, callback)`
- `raids.getBossList(callback)`

### Безопасность:
- Проверка прав администратора для всех функций
- Валидация всех пользовательских вводов
- Параметризованные SQL запросы
- Логирование всех действий

Система готова к использованию!
