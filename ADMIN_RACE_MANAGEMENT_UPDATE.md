# Обновление системы управления расами в админ панели

## Дата: 2026-04-22

## Изменения

### 1. Добавлена пагинация для выбора рас
- При редактировании рас теперь показывается список с пагинацией (5 рас на страницу)
- Добавлены стрелки навигации ◀️ ▶️ для перелистывания страниц
- Показывается текущая страница и общее количество страниц
- Расы отсортированы по редкости (DESC) и названию (ASC)

### 2. Добавлена возможность удаления рас
- В меню "Управление расами" добавлена кнопка "🗑️ Удалить расу"
- При удалении показывается список рас с пагинацией (5 рас на страницу)
- Перед удалением запрашивается подтверждение с предупреждением
- После удаления отправляется лог в канал логов
- Показывается сообщение об успешном удалении с названием расы

### 3. Улучшен интерфейс редактирования рас
- Вместо ввода названия расы теперь сразу показывается список с кнопками
- Каждая раса отображается с иконкой редкости
- Удобная навигация между страницами

### 4. Исправлен конфликт callback handlers
- Изменен обработчик удаления игрока с `confirm_delete_` на `confirm_delete_player_`
- Это предотвращает конфликт с `confirm_delete_race_` и `confirm_delete_item_`
- Теперь каждый тип удаления имеет свой уникальный префикс

## Технические детали

### Новые callback handlers:
- `admin_delete_race` - показывает список рас для удаления
- `admin_delete_race_page_X` - пагинация списка удаления
- `admin_edit_race_page_X` - пагинация списка редактирования
- `delete_race_X` - запрос подтверждения удаления
- `confirm_delete_race_X` - подтверждение и выполнение удаления

### Исправленные callback handlers:
- `confirm_delete_player_X` - удаление игрока (было `confirm_delete_X`)

### Изменения в коде:
1. **bot.js** - case 'admin_races':
   - Добавлена кнопка "🗑️ Удалить расу"

2. **bot.js** - case 'admin_edit_race':
   - Заменен ввод текста на список с пагинацией
   - Показывается 5 рас на страницу
   - Добавлена навигация стрелками

3. **bot.js** - case 'admin_delete_race':
   - Новый обработчик для удаления рас
   - Показывает список с пагинацией
   - Аналогичный интерфейс редактированию

4. **bot.js** - default section:
   - Добавлен `admin_edit_race_page_X` для пагинации редактирования
   - Добавлен `admin_delete_race_page_X` для пагинации удаления
   - Улучшен `confirm_delete_race_X` с логированием и информативным сообщением

5. **bot.js** - специальная обработка динамических данных:
   - Изменен `confirm_delete_` на `confirm_delete_player_` для избежания конфликтов
   - Теперь обработчики не пересекаются

## Логирование
При удалении расы отправляется лог:
```
🗑️ Админ удалил расу: [Название расы] (ID: [ID])
```

## Пример использования

### Редактирование расы:
1. Админ панель → 🧬 Управление расами
2. 📝 Редактировать расу
3. Выбрать расу из списка (листать стрелками если нужно)
4. Редактировать параметры

### Удаление расы:
1. Админ панель → 🧬 Управление расами
2. 🗑️ Удалить расу
3. Выбрать расу из списка (листать стрелками если нужно)
4. Подтвердить удаление
5. Раса удалена, отправлен лог

## Безопасность
- Все действия доступны только админу (проверка `config.ADMIN_USERNAME`)
- Перед удалением запрашивается подтверждение
- Показывается предупреждение о необратимости действия

## Исправленные баги
- **Баг**: При удалении расы показывалось сообщение "удален игрок id:NaN"
- **Причина**: Обработчик `confirm_delete_` перехватывал callback `confirm_delete_race_X`
- **Решение**: Переименован обработчик удаления игрока в `confirm_delete_player_X`
