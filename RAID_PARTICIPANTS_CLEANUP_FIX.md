# Raid Participants Cleanup Fix

## Исправленная проблема:

### ❌ Проблема: Список участников рейда не очищался
**Описание:** После завершения рейда список участников оставался в базе данных, что приводило к тому, что награды выдавались игрокам из предыдущих рейдов, которые не участвовали в текущем.

**Последствия:**
- Игроки получали награды за рейды, в которых не участвовали
- Список участников накапливался между рейдами
- Неправильное распределение наград по урону
- Путаница в статистике рейдов

## ✅ Решение:

### 1. **Полная очистка при создании нового рейда**
В функции `createRaidFromBoss()` добавлена полная очистка всех участников:

```javascript
// ВАЖНО: Полностью очищаем ВСЕХ участников перед созданием нового рейда
db.run(`DELETE FROM raid_participants`, (err) => {
  if (err) {
    console.error('Ошибка очистки участников рейдов:', err);
  } else {
    console.log('✅ Все участники предыдущих рейдов очищены');
  }
  // ... создание нового рейда
});
```

### 2. **Очистка при завершении рейда**
В функции `completeRaid()` добавлена очистка участников завершенного рейда:

```javascript
// Очищаем участников завершенного рейда
db.run(`DELETE FROM raid_participants WHERE raid_id = ?`, [raidId], (err) => {
  if (err) {
    console.error('Ошибка очистки участников завершенного рейда:', err);
  } else {
    console.log('🧹 Участники завершенного рейда очищены');
  }
});
```

### 3. **Очистка при провале рейда**
В функции `failRaid()` добавлена очистка участников провалившегося рейда:

```javascript
// Очищаем участников провалившегося рейда
db.run(`DELETE FROM raid_participants WHERE raid_id = ?`, [raidId], (err) => {
  if (err) {
    console.error('Ошибка очистки участников провалившегося рейда:', err);
  } else {
    console.log('🧹 Участники провалившегося рейда очищены');
  }
});
```

### 4. **Админская команда для принудительной очистки**
Добавлена новая функция `clearAllRaidParticipants()` и админская кнопка:

**В raids.js:**
```javascript
function clearAllRaidParticipants(callback) {
  console.log('🧹 Принудительная очистка всех участников рейдов...');
  
  db.run(`DELETE FROM raid_participants`, (err) => {
    if (err) {
      console.error('Ошибка очистки участников:', err);
      if (callback) callback(err);
    } else {
      console.log('✅ Все участники рейдов очищены');
      if (callback) callback(null);
    }
  });
}
```

**В bot.js (админ панель):**
- Добавлена кнопка "🧹 Очистить участников рейда"
- Обработчик `admin_clear_raid_participants` с подтверждением
- Логирование действий админа

## 🔧 Технические изменения:

3. **Обновлена `failRaid()`** - очистка участников после провала

### Файл `bot.js`:
1. **Обновлена админ панель** - добавлена кнопка очистки
2. **Добавлены обработчики** - `admin_clear_raid_participants` и `admin_confirm_clear_participants`
3. **Добавлено логирование** - запись действий админа

## 📋 Логика работы:

### Автоматическая очистка:
1. **При создании рейда**: Полная очистка всех участников
2. **При завершении рейда**: Очистка участников конкретного рейда
3. **При провале рейда**: Очистка участников конкретного рейда

### Ручная очистка:
1. **Админ команда**: Кнопка в админ панели
2. **Подтверждение**: Требуется подтверждение действия
3. **Логирование**: Запись в логи с именем админа
