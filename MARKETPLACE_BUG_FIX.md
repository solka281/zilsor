# Исправление бага дублирования предметов в маркетплейсе

## 🐛 Проблема:
Если продавец выставил 2 одинаковых предмета, то при покупке одного предмета покупатель получал 2 предмета.

## 🔍 Причины:

### 1. Неправильная логика покупки
При покупке система создавала новый предмет по `item_id` вместо передачи конкретного экземпляра.

### 2. Неправильная передача данных с фронтенда
Фронтенд отправлял `item_id` (тип предмета) вместо `inventory_id` (конкретный экземпляр).

### 3. Неправильный поиск в базе данных
Бэкенд искал предмет по `item_id` что могло найти любой предмет этого типа.

## ✅ Исправления:

### 1. Исправлена логика покупки (marketplace_api.js):
**Было:**
```javascript
// Добавляем предмет в инвентарь покупателя
db.run(`INSERT INTO inventory (player_id, item_id, equipped) VALUES (?, ?, 0)`,
  [userId, marketItem.item_id], (err) => {
```

**Стало:**
```javascript
// Переносим предмет от продавца к покупателю
db.run(`UPDATE inventory SET player_id = ?, equipped = 0 WHERE id = ?`,
  [userId, marketItem.inventory_id], (err) => {
```

### 2. Исправлена передача данных с фронтенда (marketplace/app.js):
**Было:**
```javascript
const itemId = selectedItem.item_id || selectedItem.id;
```

**Стало:**
```javascript
const itemId = selectedItem.inventory_id || selectedItem.id;
```

### 3. Исправлен поиск в базе данных (marketplace_api.js):
**Было:**
```sql
WHERE inv.player_id = ? AND inv.item_id = ? AND inv.equipped = 0
```

**Стало:**
```sql
WHERE inv.player_id = ? AND inv.id = ? AND inv.equipped = 0
```

## 🎯 Результат:
Теперь система работает правильно:
1. **Каждый предмет уникален** - используется `inventory_id`
2. **Покупка переносит предмет** - не создает дубликаты
3. **Продажа выставляет конкретный предмет** - не случайный из того же типа

## 🧪 Как протестировать:

### Шаг 1: Подготовка
1. Создайте игрока с несколькими одинаковыми предметами
2. Выставьте 2 одинаковых предмета на продажу

### Шаг 2: Тест покупки
1. Другим игроком купите один из предметов
2. Проверьте что покупатель получил только 1 предмет
3. Проверьте что у продавца остался только 1 предмет на продаже

### Шаг 3: Проверка уникальности
1. Выставьте конкретный предмет на продажу
2. Убедитесь что выставился именно тот предмет который выбрали
3. Проверьте что другие такие же предметы остались в инвентаре

## 📊 Структура данных:

### Правильный поток данных:
1. **Фронтенд:** `inventory_id` (уникальный экземпляр)
2. **Бэкенд:** Поиск по `inv.id = inventory_id`
3. **Маркетплейс:** Сохранение `inventory_id` + `item_id`
4. **Покупка:** Перенос по `inventory_id`

### Таблицы:
- `inventory.id` - уникальный ID экземпляра предмета
- `marketplace_items.inventory_id` - ссылка на конкретный экземпляр
- `marketplace_items.item_id` - тип предмета (для отображения)
