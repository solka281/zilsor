# Исправление ежедневных квестов

## Проблема
В меню квестов написано "Квесты обновятся завтра", но самих квестов нет.

## Причины
1. Таблица `daily_quests_pool` была создана, но не заполнена квестами
2. На Railway база данных уже существовала, поэтому `INSERT OR IGNORE` не добавил квесты
3. Функция `assignDailyQuests` не могла выдать квесты из пустого пула

## Решения

### 1. Улучшена инициализация (daily_quests.js)
Добавлена проверка перед инициализацией:
```javascript
db.get(`SELECT COUNT(*) as count FROM daily_quests_pool`, (err, result) => {
  if (result && result.count > 0) {
    console.log('База уже инициализирована');
    return;
  }
  // Заполняем пул квестов
});
```

### 2. Улучшена функция assignDailyQuests
Добавлены проверки и логирование:
- Проверка что пул не пустой
- Логирование ошибок
- Проверка `db.prepare` перед использованием

### 3. Добавлен скрипт проверки
Создан `check_quests.js` для проверки состояния базы:
```bash
node check_quests.js
```

## Как исправить на существующей базе

### Вариант 1: Удалить базу (потеря данных)
```bash
# На Railway
rm /app/data/game.db
# Перезапустить бота
```

### Вариант 2: Заполнить пул вручную (сохранение данных)
```bash
# Запустить скрипт заполнения
node -e "require('./daily_quests').initializeDailyQuests()"
```

### Вариант 3: Автоматическое исправление (рекомендуется)
Текущая версия автоматически проверит и заполнит пул при запуске.
Просто перезапустите бота.

## Проверка работы

1. Запустить бота
2. Проверить логи:
   ```
   Начинаем инициализацию ежедневных квестов...
   ✅ База ежедневных квестов инициализирована (100 квестов)
   ```
3. Открыть меню квестов
4. Должны появиться 3 квеста

## Логи для отладки

При проблемах проверьте логи:
```
Начинаем инициализацию ежедневных квестов...
Пул квестов пуст, заполняем...
✅ База ежедневных квестов инициализирована (100 квестов)
Выдаем 3 квестов игроку 123456789
```

Если видите:
```
Ошибка получения квестов из пула: ...
Пул квестов пуст!
```

Значит таблица не заполнена. Перезапустите бота.
