# Исправление ввода описания клана

## 🐛 Проблема:

При попытке ввести описание клана:
1. Лидер нажимает "📝 Изменить описание"
2. Вводит текст описания
3. Бот не принимает сообщение
4. Вместо этого показывает главное меню

## 🔍 Причина:

Было два обработчика `bot.on('message')`:

1. **Первый обработчик** - обрабатывает текстовые сообщения для админ-панели и состояний (adminState)
2. **Второй обработчик** - проверяет подписку и отправляет главное меню

**Проблема:** Второй обработчик перехватывал ВСЕ сообщения, включая те, которые должны обрабатываться первым обработчиком.

### Код второго обработчика (ДО исправления):
```javascript
bot.on('message', async (msg) => {
  const userId = msg.from.id;
  const chatId = msg.chat.id;
  
  // Пропускаем команды
  if (msg.text && msg.text.startsWith('/')) return;
  
  // Проверяем подписку для всех остальных сообщений
  const isSubscribed = await requireSubscription(userId, chatId);
  if (!isSubscribed) return;
  
  // Если подписан, отправляем в главное меню ❌ ПРОБЛЕМА!
  bot.sendMessage(chatId, 
    '🎮 Используйте /menu для открытия главного меню...',
    getMainMenu(true)
  );
});
```

## ✅ Решение:

Добавлены проверки во второй обработчик, чтобы он пропускал сообщения, которые должны обрабатываться первым обработчиком:

```javascript
bot.on('message', async (msg) => {
  const userId = msg.from.id;
  const chatId = msg.chat.id;
  
  // Пропускаем команды (они обрабатываются отдельно)
  if (msg.text && msg.text.startsWith('/')) return;
  
  // ✅ НОВОЕ: Пропускаем если у пользователя есть активное состояние
  if (adminState.has(userId)) return;
  
  // ✅ НОВОЕ: Пропускаем фото (обрабатываются отдельно)
  if (msg.photo) return;
  
  // Проверяем подписку для всех остальных сообщений
  const isSubscribed = await requireSubscription(userId, chatId);
  if (!isSubscribed) return;
  
  // Если подписан, отправляем в главное меню
  bot.sendMessage(chatId, 
    '🎮 Используйте /menu для открытия главного меню...',
    getMainMenu(true)
  );
});
```

## 🎯 Что изменилось:

### Добавлены две проверки:

1. **Проверка adminState:**
   ```javascript
   if (adminState.has(userId)) return;
   ```
   - Если у пользователя есть активное состояние (ввод данных), второй обработчик пропускает сообщение
   - Сообщение обрабатывается первым обработчиком

2. **Проверка фото:**
   ```javascript
   if (msg.photo) return;
   ```
   - Фото обрабатываются отдельно (аватар клана, реклама и т.д.)

✅ Изменение лимита участников клана
✅ Создание клана (ввод названия)
✅ Все админские функции с вводом текста
✅ Загрузка аватара клана (фото)
✅ Все остальные функции с adminState

## 🔄 Порядок обработки сообщений:

1. **Команды** (`/start`, `/menu` и т.д.) - обрабатываются `bot.onText()`
2. **Сообщения с adminState** - обрабатываются первым `bot.on('message')`
3. **Фото** - обрабатываются первым `bot.on('message')` или `bot.on('photo')`
4. **Остальные сообщения** - обрабатываются вторым `bot.on('message')` (проверка подписки + главное меню)
