# ⚔️ Исправление экрана боя в приключениях

## 🐛 ПРОБЛЕМЫ:

### 1. Первая атака не показывает меню
**Причина:** `startAdventureBattle` отправляла простое сообщение вместо вызова `showBattleScreen`

### 2. Неправильная картинка локации
**Причина:** `showBattleScreen` использовала `player.forest_level` вместо `battle.selectedLevel`

## ✅ ИСПРАВЛЕНИЯ:

### 1. Упрощена функция `startAdventureBattle`
**Было:**
```javascript
// Создавала свое сообщение с кнопками
bot.sendMessage(userId, message, {
  reply_markup: { inline_keyboard: keyboard }
});
```

**Стало:**
```javascript
// Использует стандартный showBattleScreen
showBattleScreen(userId, userId, null);
```

### 2. Исправлена логика выбора локации в `showBattleScreen`
**Было:**
```javascript
const forestLevel = player.forest_level || 1;
```

**Стало:**
```javascript
const battle = battleSystem.getBattle(userId);
const forestLevel = (battle && battle.selectedLevel) ? battle.selectedLevel : (player.forest_level || 1);
```

## 🎯 ЛОГИКА РАБОТЫ:

### Определение уровня для картинки:
1. **Если есть `battle.selectedLevel`** → используем выбранный уровень из приключения
2. **Если нет** → используем текущий `player.forest_level` (обычный лес)

### Выбор изображения:
1. **Определяем локацию** по уровню: `monsters.getLocationByLevel(forestLevel)`
2. **Если босс** → используем изображение босса для этого уровня
3. **Если обычный моб** → используем изображение локации
