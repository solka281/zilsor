# КРИТИЧЕСКОЕ ИСПРАВЛЕНИЕ - Краш бота при автопродаже

## ❌ Проблема
Бот крашился с ошибкой:
```
[Error: SQLITE_ERROR: cannot start a transaction within a transaction]
```

## 🔍 Причина
В функции `autoSellInventory()` в файле `auto_sell.js`:

1. Для каждого предмета вызывалась функция `sellItem()`
2. Каждый вызов `sellItem()` создавал СВОЮ транзакцию:
   ```javascript
   db.serialize(() => {
     db.run('BEGIN TRANSACTION');
     // ...
   });
   ```
3. Когда продавалось несколько предметов одновременно, возникали **вложенные транзакции**
4. SQLite не поддерживает вложенные транзакции → краш

## ✅ Решение

Переписал `autoSellInventory()` чтобы использовать **ОДНУ транзакцию** для всех предметов:

### Было (НЕПРАВИЛЬНО):
```javascript
items.forEach(item => {
  sellItem(playerId, item, (err, result) => {
    // Каждый sellItem создает свою транзакцию!
    // BEGIN TRANSACTION
    // DELETE FROM inventory...
    // UPDATE players...
    // COMMIT
  });
});
```

### Стало (ПРАВИЛЬНО):
```javascript
db.serialize(() => {
  db.run('BEGIN TRANSACTION', (err) => {
    // ОДНА транзакция для ВСЕХ предметов
    
    items.forEach(item => {
      // Удаляем предмет
      db.run(`DELETE FROM inventory WHERE id = ?`, [item.inventory_id]);
      totalGold += sellPrice;
    });
    
    // Добавляем золото ОДИН раз
    db.run(`UPDATE players SET gold = gold + ? WHERE user_id = ?`, [totalGold, playerId]);
    
    db.run('COMMIT');
  });
});
```

## 📊 Преимущества нового подхода:

1. ✅ **Нет вложенных транзакций** - одна транзакция для всех операций
2. ✅ **Атомарность** - либо продаются ВСЕ предметы, либо НИ ОДИН
3. ✅ **Производительность** - меньше операций с БД
4. ✅ **Надежность** - нет крашей при продаже нескольких предметов
