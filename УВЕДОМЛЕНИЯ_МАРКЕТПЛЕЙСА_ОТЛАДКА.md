# Исправление уведомлений маркетплейса - ОБНОВЛЕНО

## Проблемы:
1. Уведомления о покупке предметов не приходят продавцу в Telegram
2. Ошибка в аукционах: `status IN ('active', 'completed', 'cancelled')`

## Исправления:

### ✅ 1. Двойной подход к получению бота
Теперь функция `notifyItemSold()` пытается получить бота двумя способами:
- Через `global.telegramBot` (основной способ)
- Через прямой импорт `require('./bot')` (резервный способ)

### ✅ 2. Экспорт бота из bot.js
Добавлено в конец `bot.js`:
```javascript
module.exports = { bot };
```

### ✅ 3. Исправлена ошибка аукционов
Заменено:
```javascript
UPDATE auctions SET status = 'expired' WHERE id = ?
```
На:
```javascript
UPDATE auctions SET status = 'cancelled' WHERE id = ?
```

## Как протестировать:

### Шаг 1: Перезапустите сервер
Обязательно перезапустите сервер чтобы изменения применились.

### Шаг 2: Проверьте логи при запуске
Должны появиться сообщения:
```
🤖 Глобальная переменная telegramBot установлена: true
🤖 Бот запущен!
```

### Шаг 3: Протестируйте уведомления
1. Откройте `http://localhost:3000/test-notifications.html`
2. Введите свой Telegram ID
3. Нажмите "Отправить тестовое уведомление"

### Шаг 4: Проверьте реальную покупку
Попробуйте купить предмет в маркетплейсе и проверьте логи:
```
🔔 Отправляем уведомление продавцу: {...}
📨 notifyItemSold вызвана: {...}
🤖 Используем global.telegramBot (или прямой импорт)
📤 Отправляем сообщение: {...}
✅ Уведомление успешно отправлено продавцу
```

## Если все еще не работает:
1. Проверьте, что пользователь начал диалог с ботом
2. Проверьте правильность Telegram ID
3. Посмотрите на ошибки в логах
4. Убедитесь что бот имеет права на отправку сообщений
