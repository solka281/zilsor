# Система рейдов

## Дата: 2026-04-22

## Описание

Реализована полная система рейдов - совместных битв с боссами, похожая на рейды в Pokémon GO. Игроки объединяются, чтобы победить мощного босса и получить награды в зависимости от нанесенного урона.

## Основные возможности

### 1. Рейд-боссы
- 11 уникальных боссов разных уровней (1-10)
- Каждый босс имеет уникальное имя, описание и характеристики
- HP босса зависит от уровня (от 5,000 до 100,000)
- Рейд длится 2 часа

### 2. Участие в рейде
- Любой игрок может присоединиться к активному рейду
- Игроки атакуют босса своей силой
- Урон рассчитывается как сила игрока ± 20% (случайность)
- Прогресс рейда виден всем участникам

### 3. Система наград
- Минимум 2% урона для получения наград
- Награды зависят от процента нанесенного урона
- Чем больше урона - тем больше наград и шанс на предметы
- Награды: золото, кристаллы, опыт, предметы

### 4. Топ урона
- После победы показывается топ-10 игроков по урону
- Медали для топ-3: 🥇 🥈 🥉
- Каждый участник видит свою статистику

## Механика рейдов

### Создание рейда
1. При открытии меню "🐉 Рейды" проверяется наличие активного рейда
2. Если рейда нет - создается новый случайный босс
3. Рейд длится 2 часа с момента создания
4. Если время истекло - рейд проваливается

### Присоединение к рейду
1. Игрок нажимает "⚔️ Присоединиться"
2. Игрок добавляется в список участников
3. Игрок может атаковать босса

### Атака босса
1. Игрок нажимает "⚔️ Атаковать"
2. Рассчитывается урон: сила игрока × (0.8-1.2)
3. HP босса уменьшается на урон
4. Обновляется статистика игрока (урон, количество атак)
5. Если HP босса = 0 - рейд завершается

### Завершение рейда
1. Рассчитывается процент урона каждого участника
2. Участники с уроном < 2% не получают награды
3. Награды рассчитываются по формуле:
   - Базовая награда × (процент урона / 100) × множитель урона
   - Множитель урона: min(процент урона / 10, 3) - максимум ×3
4. Шанс на предмет увеличивается с процентом урона
5. Редкость предмета зависит от процента урона:
   - ≥30% - легендарный
   - ≥20% - эпический
   - ≥10% - редкий
   - <10% - обычный

## Формулы

### Урон игрока
```
урон = сила_игрока × случайность
случайность = 0.8 + random(0, 0.4) // 0.8 - 1.2
```

### Процент урона
```
процент_урона = (урон_игрока / HP_босса) × 100
```

### Множитель наград
```
множитель = min(процент_урона / 10, 3)
// Максимум ×3 для игроков с 30%+ урона
```

### Награды
```
золото = gold_min + (gold_max - gold_min) × (процент_урона / 100) × множитель
кристаллы = crystals_min + (crystals_max - crystals_min) × (процент_урона / 100) × множитель
опыт = exp_min + (exp_max - exp_min) × (процент_урона / 100) × множитель
```

### Шанс на предмет
```
шанс = min(базовый_шанс + процент_урона, 95)
// Максимум 95% шанс
```

## База данных

### Таблица active_raids
```sql
CREATE TABLE active_raids (
  id INTEGER PRIMARY KEY AUTOINCREMENT,
  boss_name TEXT NOT NULL,
  boss_level INTEGER NOT NULL,
  boss_hp INTEGER NOT NULL,
  current_hp INTEGER NOT NULL,
  boss_image TEXT,
  boss_description TEXT,
  rewards TEXT,
  started_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
  ends_at TIMESTAMP,
  status TEXT DEFAULT 'active',
  created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
)
```

### Таблица raid_participants
```sql
CREATE TABLE raid_participants (
  id INTEGER PRIMARY KEY AUTOINCREMENT,
  raid_id INTEGER NOT NULL,
  player_id INTEGER NOT NULL,
  damage_dealt INTEGER DEFAULT 0,
  attacks_count INTEGER DEFAULT 0,
  joined_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
  FOREIGN KEY (raid_id) REFERENCES active_raids(id)
)
```

### Таблица raid_history
```sql
CREATE TABLE raid_history (
  id INTEGER PRIMARY KEY AUTOINCREMENT,
  boss_name TEXT NOT NULL,
  boss_level INTEGER NOT NULL,
  total_damage INTEGER NOT NULL,
  participants_count INTEGER NOT NULL,
  status TEXT,
  completed_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
)
```

### Сценарий 1: Создание и победа над боссом

2. Создается новый рейд с боссом "Лесной тролль"
3. Игрок присоединяется к рейду

5. Другие игроки присоединяются и атакуют
6. Босс побежден
7. Все участники получают награды
8. Показывается топ урона

### Сценарий 2: Провал рейда
1. Создается рейд с боссом "Бог Разрушения"
2. Несколько игроков присоединяются
3. Игроки атакуют, но не успевают победить за 2 часа
4. Рейд проваливается
5. Никто не получает награды

### Сценарий 3: Недостаточно урона
1. Игрок присоединяется к рейду
2. Игрок атакует босса 1 раз
3. Другие игроки побеждают босса
4. Игрок нанес только 1% урона
5. Игрок не получает награды (минимум 2%)
