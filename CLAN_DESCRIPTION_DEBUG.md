# Отладка ввода описания клана

## 🔍 Добавлено логирование

Я добавил логирование в код, чтобы понять, что происходит при попытке установить описание клана.

### Что логируется:

1. **При нажатии кнопки "Изменить описание":**
   ```
   ✅ Установлено состояние для userId=XXX, action=clan_set_description, clanId=YYY
   ```

2. **При получении любого текстового сообщения:**
   ```
   📨 Получено сообщение от userId=XXX, text="...", hasState=true/false
   ```

3. **При обработке текста с активным состоянием:**
   ```
   📝 Обработка текста от userId=XXX, action=clan_set_description, text="..."
   ```

## 📋 Как проверить:

1. **Запустите бота** и откройте логи консоли
2. **Нажмите "Изменить описание"** в настройках клана
3. **Проверьте логи** - должно появиться:
   ```
   ✅ Установлено состояние для userId=...
   ```
4. **Введите текст описания**
5. **Проверьте логи** - должно появиться:
   ```
   📨 Получено сообщение от userId=..., text="...", hasState=true
   📝 Обработка текста от userId=..., action=clan_set_description, text="..."
   ```

## 🐛 Возможные проблемы:

### Проблема 1: Состояние не устанавливается
**Симптом:** Нет лога "✅ Установлено состояние..."

**Причины:**
- Игрок не является лидером клана
- Ошибка в базе данных
- Callback не срабатывает

**Решение:** Проверить права игрока и логи ошибок

### Проблема 2: Сообщение не доходит до обработчика
**Симптом:** Есть лог "📨 Получено сообщение..., hasState=false"

**Причины:**
- Состояние было удалено или не установилось
- Другой обработчик перехватил сообщение

**Решение:** Проверить, что состояние установлено и не удаляется

### Проблема 3: Обработчик не находит action
**Симптом:** Есть лог "📝 Обработка текста...", но описание не сохраняется

**Причины:**
- Ошибка в условии `else if (state.action === 'clan_set_description')`
- Ошибка в функции `setClanDescription()`

**Решение:** Проверить логи ошибок и код обработчика
